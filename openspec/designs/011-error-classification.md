## 分类详述

### H — 标题结构
- **H1 标题漏检/误检**：文章主标题未被识别为 H1（如版头机关名占位、真标题高度比不足）。修：calibrate_levels doc_title 检测（高度比/居中/文种关键词）。剩：版头占位（204号等）版头比真标题高→仍占位，待精准化。
- **H2 层级错乱**：`一、`/`（一）`/`第X章` 层级与栈序不符。修：栈式自适应定级 + `_STYLE_ORDER` + 文种关键词排序。
- **H3 文本重复**：HTML-PDF PyMuPDF 把同一标题行抽两次（bbox 一致），merge 拼接成翻倍。修：`MergeStage._is_duplicate_extraction`（同位置丢一份）。
- **H4 多行未合并**：标题跨行未合一。修：`_merge_headings`（相邻同 level 无编号合并）。
- **H5 日期当标题**：`2019年9月17日`/`2023/5/30` 被判 heading。修：`_PURE_DATE_RE` 强制段落。

### M — 段落合并
- **M1 跨页未合并**：page N 末段 + N+1 首段断开。修：`merge_cross_page`（全文档原生）。
- **M2 过度合并**：`第X条`/`1.`/`（一）` 开头的独立段被并入上一段。修：`_LIST_MARKER_RE`（标记开头不合并）。
- **M3 列表序号**：GFM `1.` 重编号丢原序号。修：OCR OL 保留 "N. xxx"，`list_markdown` 文本已含数字则原样。

### N — 噪声过滤
- **N1 页码**：未剔除。修：`filter_noise` 底部带(y1≥70%)+宽度<上方bbox×50%+正则（issues4 判据）；极端底部+序列为辅。
- **N2 页眉页脚**：盲删顶/底 N% 误删正文（R9）。修：跨页重复+layout 证据，**绝不盲删**。
- **N3 HTML 残留**：OCR `<div><img/></div>` 的开/闭标签漏网。修：纯标签块丢弃 + 内联 `<div>内容</div>` 剥离标签。

### I — 图片
- **I1 格式**：输出 `<div><img>` HTML。修：`block_markdown` GFM `![alt](id)`。
- **I2 未落盘**：PDF `_extract_page_images` 缺 mkdir。修：写前 `mkdir(image_dir)`。
- **I3 图片当段落**：盖章/页眉图未提取。修：`parse_markdown` HTML `<img>` → ImageNode（src 即 images 字典 key）。

### T — 表格
- **T1 误表**：HTML 版 PDF 首页网格被当表格。修：`_validate_table` 覆盖>50%→降级文字。
- **T2 多页表头**：长表跨页每页重复表头。修：`merge_split_tables`（连续同表头合并成一张）。
- **T3 合并单元格**：复杂表格。⏳ 表格专用模型（另一 Claude）。

### A — 附件
- **A1 附件分割**：未拆/过度拆/把引用当附件。修：纯正则全拆 + 引用清单（同块多标记）不切。🟡 剩：真附件 vs OCR 重复"附件1"的精准区分待研究。

### C — 内容/OCR
- **C1 OCR 字符**：服务侧误识。⏳ OCR 服务。
- **C2 公式**：`$ ^{**} $` 误识。⏳ OCR 服务。
- **C3 斜体**：当前无 markdown 斜体标记。✅ 非问题。
- **C4 span 序**：阅读序错乱（第五条/第二章混）。❌ 待修（PyMuPDF `_reorder_overlapping` 之外的场景）。

### V — 可视化
- **V1 漏画**：debug 可视化顶部段落被遮挡。修：header 改底线（不遮挡）。
