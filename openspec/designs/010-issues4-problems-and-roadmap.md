## 问题总账（按主题）

### 一、OCR HTML 标签残留（`</div>`）— **P0 待修**
- **影响**：9 份 OCR（2019.1.3/2023.8.22/2019.5.4/2024.9.30/2016.11.23/2018.2.24/2016.11.30/2017.12.11/2023.9.13）
- **根因**：OCR 服务把盖章/页眉图渲染成 `<div…><img/></div>`，三行被空白行拆 3 块；`_markdown.parse_markdown` 只把 `<img>` 转 ImageNode，`<div>`/`</div>` 漏网成段落。
- **方案**：`parse_markdown` 丢弃纯 HTML 标签块（`<div…>`/`</div>`/`<span…>` 等只含标签无正文）。
- **难度**：简单。

### 二、页码未除 — **P1 待修**
- **影响**：7 份（2019.1.3['4','12']/2023.8.22['334','336337']/2019.5.4/2024.9.30/2016.11.23/2018.2.24/2024.9.27）
- **根因**：`filter_noise` 页码检测要求"跨页递增序列 或 极端底部 y1≥95%"；实际页码单页/少量出现、位于 70-95% 区，不满足。
- **方案**（issues4 判据）：底部带（y1≥70% max_y）+ 宽度 < 同行/邻近最大 bbox×50% + 页码正则，**无需序列**（单页也判）。正则兼容 OCR 误识（'336337'='336/337'）。
- **难度**：中（算邻近 bbox 宽度比）。

### 三、标题问题群
- **3a 标题文本重复** — **已修**（commit 83b7e22 前）。根因：HTML-PDF PyMuPDF 重复抽取同一行（bbox 逐字节相同）。方案：`MergeStage._is_duplicate_extraction`（相邻 bbox 四坐标差<2pt + 同文本 → 丢一份）。验证 2024.2.5 重复 7→0。
- **3b 多行标题没合并成一行**（2024.10.8）— **P2 待修**。`_merge_headings` 仅覆盖跨页；同页相邻同 level 无编号标题未合并。难度：中。
- **3c 真标题漏检 + 日期当标题**（2019.9.17）— **P2 待修**。ClassificationStage 把纯日期行（`2019/8/30`、`2019年9月17日`）误判标题；真标题未识别。方案：纯日期行不判标题。难度：中。
- **3d 多级标题识别不准**（2023.8.22、2024.10.8）— **持续项**，逐例看。

### 四、附件分割 — **策略已定：所有附件都拆独立文件**
- **用户要求**：只要是附件都拆成单个文件。
- **现状**（commit 83b7e22）：`split_attachments` 纯正则全拆（heading 宽 `RE_APPENDIX` / 段落严 `RE_APPENDIX_TITLE` 排除「附件正文」）+ ④ 同块 >1 附件标记 = 引用清单不切。
- **演进**：曾试 page-top（y0<页高×25%）——对 OCR 扫描件（附件独占新页）有效（2019.1.3 11→8），但**漏掉 edited-PDF 页中附件**（流式排版接正文后，2024.12.25 等 44-86%），违背"全拆"→ 已撤为非硬门。
- **待研究**：更精准区分"真附件起始" vs "正文引用/OCR 重复'附件1'"（2019.1.3 出现 4 个不同"附件1"，疑有伪标记）。位置（页顶）+ 实质内容量判据需更多数据支撑，非硬门（避免漏拆）。
- **输出**：`batch_test` 已写 `output_附件N.md`；`assemble` 递归携带 `doc.attachments`。

### 五、有序列表重编号 — **P2 待修**
- **影响**：2019.1.3（issues4："1.xx 2.xxx 直接按原序号写入，非 markdown 重编号"）。
- **根因**：`list_markdown` 用 `1. ` marker，GFM 渲染重编号，丢失原文序号（跨页续接错乱时尤甚）。
- **方案**：列表项保留原文序号文本（"1.xxx"原样），不强制 `1.` 重编号。难度：简单偏中。

### 六、图片没保存（PDF 路径）— **P0 待修**
- **影响**：2024.2.5、2019.9.17（图存0 图引>0）。
- **根因**：`batch_test` 对 `PdfExtractor` 没传 `image_out_dir`（只 OCR 路径传了）。
- **方案**：PDF 分支也传 `image_out_dir=sub/images`。难度：简单。

### 七、其他单项 — **P3**
- **span 换行合并过多**（2025.8.29）：MergeStage 阈值偏松，结合首行缩进/句号判分段。
- **连续多页表格相同表头合并**（2023.9.13）：新功能——跨页重复表头（`|序号|事项名称|…|`）合并成一张表，引入表头栈。
- **斜体格式**（2023.8.22）：OCR 斜体标记噪声，转正文。
- **少一个附件**（2023.8.22）：附件边界漏检（与四相关）。
- **表格专用模型**：另一 Claude 在做（表格识别服务）。
