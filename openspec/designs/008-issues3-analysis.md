## 逐类详细分析

### R4：页码没过滤（12 条，最大未解决问题）

**涉及文档**：2017.13号#3, 2024.204号#2, 2023.1843号#3, 2024.59号#2, 2019.1号#1, 2025.17号#3, 2016.2号#2, 2018.30号#4, 2019.11号#3, 2016.16号#2 等

**当前状态**：
- **OCR 路线**：`parsing_res_list` 里 `page_number` 标签的块**已在映射层丢弃**（DROP_LABELS）。但如果 OCR 服务**没标** `page_number`（标成了 `text`）→ 混入 content。
- **edited-PDF 路线**：PageNumberDetection Stage 用 5 条正则 + ≥70% 页面命中。但某些格式漏网（如纯数字 "1" "2" "3" 在底部、分数式 "321/322"）。

**用户补充信息**：
- "页码在 y 轴上靠上最接近的 bbox 的宽比页码大很多" → 页码宽度远小于同行最大 bbox
- "页码应该在 70%-100%" → 页码 y 坐标在页面底部 30%
- 格式有："1"、"2"、"3"、"321/322" 等

**修复方向**：
- OCR：后处理过滤——底部 30% + 宽度 < 同行最大宽度 × 50% + 纯数字/页码正则 → 丢弃
- edited-PDF：放宽正则 + 按位置检测（底部 + 小宽度 + 序列递增）

### R1：跨页段落换行符（12 条，仍存在问题）

**涉及文档**：几乎所有 OCR 文档 + 部分 edited

**当前状态**：`join_cross_page_paragraphs` 已改为非紧邻查找（按页边界），但仍有问题。可能原因：
1. OCR 路线的 page_index 可能设置不正确（需要验证）
2. 句号检查仍太严/太松
3. 多页连续续接（跨 2+ 页）未处理

**用户补充**：建议"通过句子没遇到句号就遇到换行符这种不合理现象判断" → 如果文本中没句号就有换行，可能是跨页断句。

**涉及文档**：2024.59号, 2019.1号, 2019.5.4, 2023.68号, 2026.38号, 2017.13号, 2018.30号 等

**用户反馈**：

**需验证**：当前 `export/_helpers.py` 的 ImageNode → `![alt](image_id)` 格式是否被另一 session 改成了 HTML。issues2 已确认当时是 GFM 格式，但 issues3 又报告——说明中间被改过。

### R2：OCR 文章标题没识别（8 条）

**涉及文档**：2017.13号, 2024.204号, 2023.1843号, 2024.59号, 2019.1号, 2016.2号, 2018.30号, 2026.38号, 2023.68号 等

**当前状态**：
- calibrate 的 `DOC_TITLE_RATIO` 检测（ratio > 1.8）在 OCR 路线有效
- 但部分文档的标题 bbox 高度与正文差异不够大 → ratio < 1.8 → 未检测
- 已实现 doc_title fallback（首个无编号 H1/H2 heading → doc_title），但 OCR 标题往往没被 ClassificationStage 判为 heading（OCR 走 `_classify_ocr` 不走多信号）

**核心矛盾**：OCR 路线的 ClassificationStage（`_classify_ocr`）只看版面 `title` 标签，不走多信号。如果服务没标 `title`，标题被当 paragraph → calibrate 的 doc_title 检测拿不到 heading 级别的 title → fallback 也触发不了。

**修复方向**：OCR 路线也用多信号（编号 + 高度 + centered）做 heading 判定，或 calibrate 对 OCR 增强大标题检测。

### R10：段落合并不分（3 条）

**涉及文档**：2025.17号（所有内容混一段）, 2022.129号, 2024.2.5号

**用户关键信息**（2025.17号）："每一条的管理办法没有换行符" → 第X条法律条文，每条应独立段落。
**用户关键信息**（2022.129号）："上下行间距确实不够宽，也不可能够，因为他们是正文内容" → 不能只靠间距。

**修复方向**：
- 加入首行缩进检测（新段落有缩进，续行没有）
- 加入编号开头检测（第X条、X. → 新段落/列表项）
- 前一行末尾句号检测
